In my initial post I highlighted some differences between intelligent agents and objects and then described why Agent Communication Languages (ACLs) such as Knowledge Query and Manipulation Language (KQML) are useful, especially with intelligent agent architectures involving multicast and broadcast connectivity with mainly asynchronous communication (Addakiri & Bahaj, 2013).

I went on to note that disadvantages include deadlock, where two agents try to start a communication with each other and neither cede (Hai-long & Wu, 2001), and sequencing problems in asynchronous solutions due to there being no obligation for an agent to respond to or acknowledge a message (Brooks et al, 1998).

Van Toor (2023b) found a solution to deadlocks using petri nets to model and monitor resources that could have contention and through siphons and traps ensure that resources are allocated (Čapkovič, 2023). I challenged that this solution relies upon centrally monitored resources, and so probably wouldn’t work in a multi-agent system with fully independently agents. Van Toor (2023b) came back with two further papers, Kuo & Lin (2006) and Celaya et al. (2007), which both used petri nets in independent multi-agent systems. Whilst they were not being used specifically to prevent deadlocks, it does seem to suggest that there is potential in the solution proposed by Čapkovič (2023) even for an independent agent architecture.

Lukashevich (2023a) highlighted a solution using petri nets for workflow modelling in multi-agent systems whereby if a change is detected the agents can be instructed to break contracts with other agents and then re-contract as required to achieve the goals. This further highlights that I was hasty with my initial scepticism.  

Finally, I clarified in Feaviour (2023a) and Feaviour (2023b) that ACLs are most useful in multi-agent systems with autonomous agents, which would not be considered a distributed system. Lukashevich (2023b) insightfully shared how game theory can help agents to negotiate with each other (Darrow, 2023) which I found was a helpful way to differentiate from a distributed system.
