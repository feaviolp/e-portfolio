Machine Learning

Machine learning with a practical approach, involving hands-on cases of real-world data analytics.

Learning Outcomes

  1. Articulate the legal, social, ethical and professional issues faced by machine learning professionals.
  2. Understand the applicability and challenges associated with different datasets for the use of machine learning algorithms.
  3. Apply and critically appraise machine learning techniques to real-world problems, particularly where technical risk and uncertainty is involved.
  4. Systematically develop and implement the skills required to be effective member of a development team in a virtual professional environment, adopting real-life perspectives on team roles and organisation.

Reflections

Unit 1
The lecturecast "Introduction to Machine Learning" provided a really good introduction to many different topics to be covered in the module. This had the greatest depth of information of any lecturecast so far, and it was only the introduction. The videos were well paced and it has certainly made me curious about the rest of the module.

The first sub-chapter of the core book "An Introduction to Machine Learning" introducted why machine learning is needed without yet getting into the complexities of how to implement it while the online reading gave a bit of history of machine learning. There was a paper on correlation and regression that described some statistics used in machine learning, and whilst I understood the concepts I didn't attempt to work through the equations. Finally, the paper "Deep Learning in Social Computing" gave an interesting insight into social science and computer science come trogehter. It did, however, remind me of the Cambridge Analytica scandal and the ethical considerations of using technology for social engineering.

There was a collaborative discussion about the impact of industry 4.0. I used the example of Boeing 737 MAX where the reckless use of technology caused two plane crashes. My initial post can be found here:

  • Initial Post

Unit 2
The reading for unit 2 was very short and simple, covering the concepts of machine learning. That was luck because the activities this week were very time consuming.

Exploratory Data Analysis was performed in Python using a provided Jupyter notebook on a sample dataset, in this case relating to house prices. The code was then to be modified to analyse a different dataset concerning miles per gallon of different cars. The chalenge was I didn't have Python, and IDE or Jupyter notebook installed. I tried installing them on my work laptop with limited success due to security policies in place, so I switched to my personal laptop. Here I managed to get them installed, using Pycharm as the IDE after trying unsuccessfully to get Visual Studio Code to work with the correct Python interpreter, but even with the sample code I faced many errors and warnings. After a very long time fault finding I eventually got it to work, albeit with some warnings remaining. Incidentally there were still errors and warnings when running on the Codio environement provided by Essex university, so it seems that the provided code includes some depricated functionality. When I finally got it all working there were some very useful features in the analysis. I'd like to put on record my thanks to Astrid van Toor for helping me to setup Pycharm when I was facing issues due to having had so many attemtmps with remnants of Python versions including and excluding Anaconda.
